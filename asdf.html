Flutter × Isar
講義登録フォームの構築

直感的なUIと堅牢なデータ保存の両立

UI & Data Binding

ユーザーの入力をIsarモデルへと橋渡しする設計

必要な入力項目

1. 基本情報

講義名（名前）、教室名、担当教員など。`TextFormField`を使用してバリデーションを実装します。

2. 管理設定

許容欠席回数（デフォルトは5回など）。`Dropdown`やスライダーで直感的に選択可能にします。

フォームの主要コンポーネント

TextField

講義名入力用。`InputDecoration`でラベルを表示。

Counter UI

欠席限度回数の増減ボタン。タップミスを防ぐ設計。

Save Button

Isarの`writeTxn`を呼び出すトリガー。

ユーザー体験の最適化

フォーム入力はユーザーにとって最も負荷のかかる作業です。オートフォーカスや、過去の入力履歴からのサジェスト機能を実装することで、登録時間を大幅に短縮できます。


特に「欠席限度」などの数値入力は、キーボードを開かずに増減できる独自ウィジェットが効果的です。

実装コードの要点

Controllerの活用

`TextEditingController`を使用して、入力内容をリアルタイムに取得します。


保存処理

「保存」ボタン押下時に、入力値を`Subject`クラスに代入し、`isar.subjects.put(subject)`を実行。非同期処理による待機アニメーションも重要です。

UIとIsarモデルの対応マップ

UIウィジェット Isarモデルフィールド データ型 備考
TextFormField name String 必須入力・重複不可
NumberPicker limitCount int デフォルト: 5
Switch / Checkbox isAutomatic bool 自動出席のON/OFF

利便性の数値化

1.5sec

瞬時の登録完了

最適化されたUIにより、講義1つの登録にかかる時間はわずか1.5秒。事前の時間割登録を苦にさせないスピード感を提供します。

実装の4ステップ

  • FormKeyの設定: `GlobalKey `を使用してフォーム全体の状態を管理。
  • バリデーション: 「名前が空ではないか」「数値が負ではないか」をチェック。
  • トランザクション実行: `writeTxn`内で安全にデータを永続化。
  • フィードバック: `SnackBar`等で保存完了をユーザーに通知。

出席率の計算式と可視化

Attendance Rate = Attended Classes Total Registered Classes × 100 %

登録されたデータに基づき、学期末の予測出席率をグラフ化。警告ラインを下回らないよう管理を徹底します。
